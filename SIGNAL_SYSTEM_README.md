# ETF调仓信号抓取系统使用说明

## 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                        数据采集层                            │
├─────────────────┬──────────────────┬──────────────────────┤
│   Tushare Pro   │    新浪财经       │     东方财富         │
│  (major_news)   │   (网页抓取)      │    (网页抓取)        │
│    每2小时       │     每日          │      每日            │
└─────────────────┴──────────────────┴──────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│                        信号分析层                            │
├─────────────────────────────────────────────────────────────┤
│  关键词匹配 → 匹配度计算 → 信号生成 → 待复核队列            │
│  - 地缘冲突关键词                                          │
│  - 科技轮动关键词                                          │
│  - 宏观数据关键词                                          │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│                        人工复核层                            │
├─────────────────────────────────────────────────────────────┤
│  人工审核 pending_signals → 批准/拒绝/修改 → 发布           │
│  高优先级信号立即邮件通知                                    │
└─────────────────────────────────────────────────────────────┘
```

## 定时任务

| 任务 | 时间 | 频率 | 说明 |
|:---|:---|:---:|:---|
| 信号自动抓取 | 09:00, 15:00 | 交易日 | 自动抓取新闻并生成候选信号 |
| 信号复核提醒 | 09:30, 15:30 | 交易日 | 如有pending信号，邮件提醒复核 |
| 过期信号清理 | 每日00:00 | 每日 | 自动清理过期活跃信号 |

## 使用方法

### 1. 查看待复核信号

```bash
cd /root/.openclaw/workspace/fund_data/analysis
python3 scripts/review_signals.py list
```

### 2. 交互式复核

```bash
python3 scripts/review_signals.py review
```

交互选项：
- `a` - 批准全部
- `r` - 拒绝全部
- `数字` - 复核单个信号（如输入 1 复核第一个）
- `q` - 退出

### 3. 批准/拒绝单个信号

```bash
python3 scripts/review_signals.py approve AUTO_20260314_001
python3 scripts/review_signals.py reject AUTO_20260314_001
```

### 4. 手动抓取信号

```bash
# 设置Tushare Token（可选）
export TUSHARE_TOKEN="your_token"

# 运行抓取
python3 scripts/fetch_signals_auto.py
```

## 信号数据文件

- `etf_signals.json` - 信号主文件
  - `active_signals` - 已发布的活跃信号
  - `pending_signals` - 待人工复核的信号
  - `signal_rules` - 信号规则配置

## 注意事项

1. **Tushare Token**: 如需使用Tushare抓取，需设置环境变量 `TUSHARE_TOKEN`
2. **反爬限制**: 新浪财经/东方财富可能有反爬，建议抓取频率不要过高
3. **人工复核**: 所有自动生成的信号默认进入pending队列，需人工复核后发布
4. **信号有效期**: 活跃信号默认3天过期，过期后自动清理

## 故障排查

### 抓取失败

```bash
# 测试单个数据源
python3 -c "from scripts.fetch_signals_auto import fetch_sina_finance; print(fetch_sina_finance())"
```

### 邮件发送失败

检查139邮箱授权码是否正确（当前使用TOOLS.md中配置的授权码）

### 信号未生成

检查关键词匹配度阈值是否过高，或新闻内容是否包含关键词
