### 现状
- 已安装 tmux skill，但很少使用
- 主要用 exec 和 process 工具管理任务
- 不熟悉 tmux 的高级功能

### 学习目标
1. 掌握 tmux 基础操作（创建、切换、关闭会话）✅
2. 学习长任务管理（后台运行、监控、恢复）✅
3. 了解多窗格并行操作

### 实践成果
**核心命令掌握：**
```bash
# 创建会话
SOCKET="${TMPDIR:-/tmp}/openclaw-tmux-sockets/openclaw.sock"
tmux -S "$SOCKET" new -d -s <session-name> -n <window-name>

# 发送命令
tmux -S "$SOCKET" send-keys -t <session>:<window>.<pane> -- '<command>' Enter

# 捕获输出
tmux -S "$SOCKET" capture-pane -p -J -t <session>:<window>.<pane> -S -200

# 列出/关闭会话
tmux -S "$SOCKET" list-sessions
tmux -S "$SOCKET" kill-session -t <session-name>
```

**使用场景：**
- 长任务后台运行（不会被中断）
- 交互式 CLI 控制
- 多任务并行执行
- 持续监控输出

**三种摘要方法掌握：**

| 方法 | 特点 | 适用场景 |
|:---|:---|:---|
| **提取式** | 原文引用，最准确 | 需要引用论证时 |
| **生成式** | 改写概括，最流畅 | 需要深度理解时 |
| **结构化** | 表格呈现，最实用 | 快速检索对比时 |

**质量评估维度：**
- 准确性（是否忠实原文）
- 简洁度（信息密度）
- 可读性（理解成本）
- 可操作性（能否指导行动）

- 对韦冀星研报进行了三种方法的摘要

```
问题定义 → 多源搜索 → 交叉验证 → 信息整合 → 结论输出
```

**交叉验证维度：**
- 同一判断的多源确认
- 分歧点的识别与标记
- 一致性强度评估（强共识/分歧/待验证）

**实践案例**：`research-practice-20260320.md`
- 主题："安全定价"范式转移
- 信息源：4位分析师/机构

### 关键技巧

| 技巧 | 说明 | 应用场景 |
|:---|:---|:---|
| **多源对比表** | 同一维度横向对比不同来源 | 识别共识与分歧 |
| **逻辑链图示** | 从宏观到微观的因果推导 | 理清复杂议题 |
| **行动优先级矩阵** | 按重要性和紧急性排序 | 指导投资决策 |
| **质量自评清单** | 研究完成后的自我检查 | 持续提升质量 |
