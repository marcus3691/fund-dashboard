# 基金产品库定期更新方案
## 筛选表现前10%基金并深度研究入库

---

## 一、方案概述

**目标：** 建立自动化/半自动化的基金产品库，定期筛选市场前10%表现基金，进行深度分析后入库，形成可追踪、可复用的产品研究档案。

**核心价值：**
- 发现优秀基金经理和产品
- 建立产品研究知识库
- 支持投资决策和组合配置
- 追踪产品表现变化

---

## 二、频率建议

### 推荐方案：双周滚动 + 季度深度

| 频率 | 任务 | 工作量 | 说明 |
|:---|:---|:---:|:---|
| **双周** | 数据更新+初步筛选 | 2-3小时 | 更新净值数据，识别新进入前10%的基金 |
| **月度** | 快速分析报告 | 4-6小时/只 | 对新进入前10%的基金进行快速分析 |
| **季度** | 深度研究入库 | 8-12小时/只 | 对持续表现优秀的基金进行深度分析并正式入库 |

### 为什么不选更高频率？

| 频率 | 问题 |
|:---|:---|
| 每日 | 净值数据有延迟，且短期波动噪音大 |
| 每周 | 工作量过大，且周度排名不稳定 |
| 每月（仅月度）| 可能错过季度调仓的关键节点 |

### 为什么不选更低频率？

| 频率 | 问题 |
|:---|:---|
| 半年度 | 错过中期表现优秀的产品 |
| 年度 | 年度排名已反映到价格中，失去先发优势 |

---

## 三、分析区间建议

### 核心推荐：多维度区间评估

不要只看单一区间，应该同时考察多个时间维度：

| 分析区间 | 权重 | 适用场景 | 说明 |
|:---|:---:|:---|:---|
| **近6个月** | 30% | 短期动量、调仓效果 | 反映最新投资能力，但噪音较大 |
| **近1年** | 40% | 年度考核、相对排名 | 最常用区间，夏普比率有意义 |
| **近3年** | 20% | 长期能力、穿越周期 | 验证策略可持续性 |
| **成立以来** | 10% | 历史业绩、长期复利 | 参考，但受成立时点影响大 |

### 具体执行方案

### 分类筛选标准（关键：按类别分别筛选前10%）

**基金分类：**
| 类别 | 说明 | 前10%数量估算 |
|:---|:---|---:|
| **主动股票型** | 股票仓位>80%的主动基金 | ~80只 |
| **混合型-偏股** | 股票仓位60-80% | ~150只 |
| **混合型-平衡** | 股票仓位40-60% | ~100只 |
| **混合型-偏债** | 股票仓位<40% | ~50只 |
| **FOF** | 基金中基金 | ~10只 |
| **指数增强型** | 主动管理+指数跟踪 | ~30只 |
| **债券型-主动** | 主动债券管理 | ~50只 |
| **QDII** | 海外投资基金 | ~20只 |

**双周筛选标准（按类别分别筛选）：**
- 在**同类别**中，近6个月收益排名前10%
- 在**同类别**中，近1年收益排名前15%
- 波动率 < 该类别的75%分位数（排除极端）

**季度入库标准（按类别精选）：**
- 在**同类别**中，近1年收益排名前10%
- 在**同类别**中，近6个月持续保持前20%
- 夏普比率 > 同类别中位数
- 最大回撤 < 同类别的75%分位数
- 基金经理任职 > 1年

**为什么必须按类别筛选？**
- 股票型和债券型收益特征完全不同，混在一起排名无意义
- FOF和主动股票型风险收益特征不同
- 同类别对比才能真实反映基金经理的相对能力

## 四、产品库结构设计

### 产品库分级

| 级别 | 标准 | 分析深度 | 更新频率 |
|:---|:---|:---:|:---:|
| **核心库** | 连续2个季度前10% | 深度 | 季度跟踪 |
| **观察库** | 首次进入前10% | 标准 | 月度跟踪 |
| **备选库** | 曾进入前10%但目前退出 | 快速 | 双周监控 |

### 产品档案内容

每只入库基金包含：
1. **基本信息** - 代码、名称、类型、经理、规模
2. **业绩数据** - 各区间收益、排名、波动率、夏普
3. **持仓分析** - 季度持仓演变、行业分布、重仓股
4. **深度分析** - 9维度AI分析
5. **同类对比** - 与波动相近基金对比排名
6. **跟踪记录** - 入库后表现变化、调仓跟踪

## 五、工作流程设计

### 双周工作流程（自动化+人工）

```
周一上午：Cron自动任务
  └─ 更新全市场基金净值数据
  └─ 计算各区间收益排名
  └─ 筛选进入前10%的新基金
  └─ 生成候选名单

周一下午：人工确认
  └─ 查看候选基金名单
  └─ 剔除异常基金（如巨额赎回导致的收益异常）
  └─ 确认进入"观察库"
```

### 月度工作流程

```
每月第一周：
  └─ 对"观察库"基金进行快速分析
  └─ 生成1-2页简报
  └─ 判断是否有必要深度研究
```

### 季度工作流程

```
季末后2周：
  └─ 对连续2季度优秀的基金进行深度分析
  └─ 执行完整SOP（9维度分析）
  └─ 正式纳入"核心库"
  └─ 生成完整PDF报告
```

## 七、预期工作量评估

### 完整的入库指标体系

**一级指标（必须满足）：**
| 指标 | 筛选标准 | 说明 |
|:---|:---|:---|
| **同类排名** | 近1年收益排名前10%（同类别） | 核心指标，必须满足 |
| **持续性** | 近6个月排名保持前20%（同类别） | 排除短期运气 |
| **基金经理** | 任职时间 > 1年 | 确保策略稳定性 |

**二级指标（质量评估）：**
| 指标 | 优秀标准 | 良好标准 | 观察标准 |
|:---|:---|:---|:---|
| **夏普比率** | > 同类别75%分位 | > 同类别中位数 | > 同类别25%分位 |
| **最大回撤** | < 同类别25%分位 | < 同类别中位数 | < 同类别75%分位 |
| **年化波动率** | < 同类别25%分位 | < 同类别中位数 | < 同类别75%分位 |
| **月度胜率** | > 70% | > 60% | > 50% |
| **盈亏比** | > 2.0 | > 1.5 | > 1.0 |

**三级指标（深度评估）：**
| 指标 | 用途 | 数据来源 |
|:---|:---|:---|
| 换手率 | 评估调仓频率 | 半年报/年报 |
| 行业集中度 | 评估风险分散度 | 季报持仓 |
| 重仓股稳定性 | 评估持股周期 | 季报持仓对比 |
| 规模变化 | 评估容量影响 | 定期公告 |
| 机构持有比例 | 评估专业认可度 | 半年报/年报 |

**一票否决项（出现即不入库）：**
- ❌ 基金经理变更（任职 < 6个月）
- ❌ 巨额赎回（单日净值异常波动 > 5%）
- ❌ 违规记录（监管处罚、信息披露问题）
- ❌ 规模过小（< 1亿元，清盘风险）
- ❌ 波动率过高（> 同类别90%分位）

**入库漏斗（每双周）：**
```

    ↓ 二级指标评估（质量评分）

    ↓ 快速分析/深度研究
月度分析 ~5-8只
    ↓ 季度深度+入库
季度入库 ~3-5只
```

## 八、风险控制

### 筛选偏差风险

| 风险 | 说明 | 应对措施 |
|:---|:---|:---|
| **幸存者偏差** | 只关注表现好的，忽略已退出的 | 同时追踪"曾入库但退出"的基金 |
| **短期运气** | 6个月排名可能靠运气 | 要求连续2季度前10%才入库 |
| **风格漂移** | 基金经理风格变化 | 季度深度分析时重点检查 |
| **规模效应** | 规模变大后策略失效 | 监控规模变化，设置上限 |

### 数据质量风险

- 净值数据延迟：Tushare通常T+1，部分基金T+2
- 持仓数据滞后：季报滞后15-30天
- 应对措施：数据更新时标注延迟天数

## 九、具体执行建议

### 第一阶段（1-2个月）：试运行

1. **先执行1-2轮双周筛选**，熟悉流程
2. **选择5-10只基金进行深度分析**，验证SOP可行性
3. **建立产品库基础结构**，确定档案格式
4. **评估实际工作量**，调整频率和范围

### 第二阶段（3-6个月）：正式运行

1. **完善自动化脚本**，减少人工操作
2. **建立核心库**（目标20-30只基金）
3. **建立观察库**（目标50-80只基金）
4. **形成定期报告机制**

### 第三阶段（6个月后）：优化迭代

1. **回溯验证**：入库基金后续表现如何？
2. **优化筛选标准**：调整权重和阈值
3. **扩展覆盖**：增加债券型、QDII等
4. **建立预警机制**：入库基金表现异常时提醒
