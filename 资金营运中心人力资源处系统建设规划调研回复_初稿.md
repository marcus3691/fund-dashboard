# 资金营运中心人力资源处系统建设规划调研回复（初稿）

---

## 一、业务规划内容

### 1. 主要业务职责及业务范围

资金营运中心人力资源处主要负责本行企业年金计划受托管理，具体包括：

**（1）本行年金日常受托运营**
- 员工年金账户管理（新增、变更、状态转换、转入、转出等）
- 缴费管理：年金计划下各机构月度缴费的归集、核对与处理
- 支付管理：退休、离职等情形下的年金待遇支付申请审核与资金划转
- 转移管理：跨机构、跨计划的年金权益转移处理
- 投资划款指令管理：定价日申购赎回指令的编制与执行

**（2）受托投资管理**
- 年金投资组合的监督与管理
- 投资管理人、托管人的履职监督
- 年金组合费用（管理费、托管费、业绩报酬）的核算与支付
- 投资业绩计量与归因分析
- 投资风险监控与合规管理

**主要产品**：本行企业年金计划受托管理服务

### 2. 中长期（3-5年）业务规划

围绕"电子化、自动化、智能化"三大方向，推进以下重点工作：

| 方向 | 具体规划内容 | 目标时间 |
|------|-------------|---------|
| **系统直连** | 推进企业年金受托管理系统与人力资源管理系统、外部账户管理系统的业务和数据直连传输 | 2025-2026年 |
| **委托人体验优化** | 开放员工年金投资组合自主选择权，提升员工福利保障服务能力 | 2026年 |
| **投资风险管理提升** | 引入模型加工实现组合安全垫测算和监控，完善投资经理评价体系，建立长周期考核机制 | 2025-2026年 |
| **数据整合** | 引入投研和信评数据、市场行情数据，优化大类资产配置决策支持 | 2025年 |
| **受托户核算能力建设** | 构建完整的受托户财务核算体系，实现凭证、账表、期末处理、报表出具等核心功能 | 2025-2026年 |

### 3. 行内战略落实规划

针对全行工作会议及五年发展战略规划的相关要求，处室落实规划如下：

| 战略要求 | 落实规划 |
|---------|---------|
| FICC产品第二曲线 | 企业年金投资范围已覆盖债券、股票、基金、养老金产品等，将持续优化资产配置结构 |
| 三大海外中心跨时区交易 | 年金投资以境内资产为主，暂不涉及海外交易 |
| 更精细的客户沙盘、细分客群和重点场景 | 推进员工投资组合自主选择权开放，提升个性化服务能力 |
| 从债券向资金/外汇/大宗商品/衍生品领域延伸 | 在合规框架内，适时研究拓展年金投资品种 |
| 保持汇率报价优势 | 年金投资以人民币资产为主，汇率风险敞口有限 |
| 保持积存金业务优势 | 积极探索贵金属类养老金产品创新 |
| 丰富贵金属产品 | 研究将贵金属纳入年金投资范围的可行性 |
| 落实"1+7"AI业务蓝图建设人工智能标杆场景 | 在年金受托管理领域探索AI智能应用场景（详见第四部分） |

## 二、能力规划内容

### （1）产品能力

**现状不足**：
- 目前仅提供单一的企业年金受托管理服务，产品类型相对单一
- 缺乏面向不同风险偏好员工的差异化产品选择

**改进计划**：
- 推进员工投资组合自主选择权开放，丰富产品供给
- 研究探索贵金属类养老金产品创新

### （2）客户服务能力

**现状不足**：
- 员工个人权益查询渠道有限，主要依赖线下或邮件查询
- 移动端服务能力不足

**改进计划**：
- 建设移动端业务申请收集和权益查询功能
- 优化委托人服务体验，提升员工福利保障服务能力

### （3）渠道服务能力

**现状不足**：
- 业务申请主要依赖线下或PC端，移动端覆盖不足
- 多渠道协同能力有待提升

**改进计划**：
- 推进员工作业平台待办任务优化，实现批次汇总展示
- 建设移动端业务申请收集功能

### （4）投资交易能力

**现状不足**：
- 投资划款指令处理存在一定手工操作
- 转移指令推送效率有待提升，大批量划款后反馈回盘存在堵塞

**改进计划**：
- 优化转移指令推送效率，参照支付指令批次反馈机制
- 推进投资交易流程自动化

### （5）投资决策能力

**现状不足**：
- 投前决策支持工具相对有限
- 投后组合分析和策略执行监测能力有待加强
- 缺乏系统化的投资经理评价体系

**改进计划**：
- 建设投资经理分析模块，实现投资经理打标、业绩曲线拟合、深度分析和市场对比评价
- 引入均值方差优化模型等先进资产配置方法
- 建设组合安全垫测算功能，强化风险监控

### （6）风险合规能力

**现状不足**：
- 组合风险监控工具相对单一，缺乏动态安全垫测算能力
- 投资合规规则引擎有待完善

**改进计划**：
- 建设组合安全垫测算功能（静态+动态），实现组合不同口径安全垫测算和监控
- 完善投资监督功能，强化合规管理

### （7）数据应用能力

**现状不足**：
- 部分数据分析仍依赖手工Excel处理
- 数据整合能力有待提升，内外部数据融合不足
- 大表查询性能存在瓶颈（如持仓明细表3000万+记录）

**改进计划**：
- 推进投研系统建设，实现数据分析全流程系统化
- 引入行内投研和信评数据、市场行情数据，拓宽数据来源
- 优化系统性能，解决慢SQL问题，支持跨月下载

### （8）业务连续性能力

**现状不足**：
- 电子档案管理尚未完全线上化，业务审批单及附件缺少系统备份

**改进计划**：
- 对接电子档案系统，实现业务审批单自动归档和在线查阅

### （9）协同联动能力

**现状不足**：
- 与人力资源系统的数据交互范围有限
- 与外部账户管理系统的直连尚未建立

**改进计划**：
- 扩展与人力资源系统的数据对接，实现转入、转移业务自动化
- 推进与外部账户管理系统的直连对接

### （10）其他能力

**受托户财务核算能力**：
- 现状：受托户核算模块目前依赖外部系统，尚未建立自研改造
- 改进：建设受托户核算模块，包括业务公共、基础数据、财务会计三大子模块，实现凭证管理、账表查询、期末处理、报表出具、会计平台等核心功能

## 三、在用系统使用意见

### 在用系统清单

| 序号 | 系统名称 | 主要用途 |
|------|---------|---------|
| 1 | 企业年金受托管理系统 | 年金日常受托运营（员工管理、缴费、支付、转移等） |
| 2 | 投研平台-年金投资分析模块 | 年金投资组合管理、投资分析、业绩归因、资产配置测算 |

### 现有系统问题与不足

**企业年金受托管理系统**：
1. **业务流程方面**：分支机构业务仍采用单人经办模式，缺少复核环节；超管岗位权限过于集中
2. **系统集成方面**：尚未与电子档案系统、Gut供数平台完全对接
3. **性能方面**：部分大表查询存在慢SQL问题，跨月下载功能受限
4. **功能完善方面**：缴费管理缺少预缴费核对功能，费用管理缺少试算功能
5. **财务管理方面**：受托户核算模块依赖外部系统，自研改造尚未完成

**投研平台-年金投资分析模块**：
1. **分析维度不足**：主要支持组合维度分析，缺乏投资经理维度分析
2. **收益分解粒度不够**：场外基金、ETF基金、养老金产品收益分析功能待完善
3. **资产配置模型单一**：需要引入均值方差优化等先进模型
4. **缺乏投资经理评价体系**：长周期考核机制落地缺乏系统支撑
5. **缺乏安全垫测算工具**：组合风险监控能力有待加强

### 意见建议

1. 加快企业年金受托管理系统三期建设，重点解决流程优化、系统集成、性能提升和受托户核算能力建设
2. 推进投研系统二期建设，完善投资经理评价、组合安全垫测算等核心功能
3. 加强两个系统之间的数据协同，构建统一的企业年金业务处理平台

| 场景领域 | 具体应用方向 |
|---------|-------------|
| **智能交易** | 探索基于规则引擎的智能投资划款指令生成 |

## 五、对标同业机构

### 1. 对标机构清单

| 机构类型 | 机构名称 |
|---------|---------|
| 国有行 | 工商银行、建设银行 |
| 股份行 | 招商银行 |
| 专业养老金公司 | 平安养老险、太平养老险 |

### 2. 对标优势

同业机构在以下方面具有领先优势：
- 投资经理评价体系完善，支持长周期考核机制
- 风险管理工具丰富，具备组合安全垫监控、风险预警等功能
- 资产配置模型先进，广泛应用均值方差优化、Black-Litterman模型等
- 数据整合能力强，能够整合内外部数据源实现多维度投研分析

### 3. 可借鉴经验

希望借鉴同业机构在投资经理评价、组合风险监控、资产配置优化等方面的功能设计和流程架构，提升本行年金投资管理的专业化水平。

## 七、部分内容收集

### 1. 手工台账和处理情况

**日常运营受托业务记录台账**：
- 支付申请、变更申请台账
- 缴费业务台账
- 转入业务台账
- 投资划款指令台账

**受托投资管理相关台账**：
- 年金组合费用台账
- 年金组合实际收益率台账
- 非标资产台账
- 风险管理台账
- 周度/月度/季度受托投资材料数据台账

**系统改造需求**：上述台账中，除周度/月度/季度投资材料数据台账涉及复杂数据分析外，其余台账均可在受托管理系统三期和投研系统二期建设后实现线上化。

### 2. 移动端审批需求

- 员工个人权益查询
- 业务申请提交（支付申请、转移申请等）
- 审批流程待办处理

### 3. 数据需求

**现有数据满足情况**：
- 行内人力资源数据：基本满足，需扩展转入、转移等业务的数据对接
- 投资管理数据：基本满足，需引入更多市场数据

**未满足情况**：
- 外部市场行情数据需进一步丰富
- 投研和信评数据需加强整合
