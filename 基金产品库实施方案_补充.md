# 基金产品库实施方案补充
## 数据库字段设计与启动流程

## 一、完整的入库指标体系

### 一级指标（必须满足）
| 指标 | 筛选标准 | 说明 |
|:---|:---|:---|
| **同类排名** | 近1年收益排名前10%（同类别） | 核心指标，必须满足 |
| **持续性** | 近6个月排名保持前20%（同类别） | 排除短期运气 |
| **基金经理** | 任职时间 > 1年 | 确保策略稳定性 |

### 二级指标（质量评估）
| 指标 | 优秀标准 | 良好标准 | 观察标准 |
|:---|:---|:---|:---|
| **夏普比率** | > 同类别75%分位 | > 同类别中位数 | > 同类别25%分位 |
| **最大回撤** | < 同类别25%分位 | < 同类别中位数 | < 同类别75%分位 |
| **年化波动率** | < 同类别25%分位 | < 同类别中位数 | < 同类别75%分位 |
| **月度胜率** | > 70% | > 60% | > 50% |
| **盈亏比** | > 2.0 | > 1.5 | > 1.0 |

### 三级指标（深度评估）
| 指标 | 用途 | 数据来源 |
|:---|:---|:---|
| 换手率 | 评估调仓频率 | 半年报/年报 |
| 行业集中度 | 评估风险分散度 | 季报持仓 |
| 重仓股稳定性 | 评估持股周期 | 季报持仓对比 |
| 规模变化 | 评估容量影响 | 定期公告 |
| 机构持有比例 | 评估专业认可度 | 半年报/年报 |

### 一票否决项
- ❌ 基金经理变更（任职 < 6个月）
- ❌ 巨额赎回（单日净值异常波动 > 5%）
- ❌ 违规记录（监管处罚、信息披露问题）
- ❌ 规模过小（< 1亿元，清盘风险）
- ❌ 波动率过高（> 同类别90%分位）

## 三、数据库准备工作流程

### 阶段1：基础数据采集（数据库完成后立即开始）
- [ ] 全市场基金列表获取（~4,900只）
- [ ] 基础信息录入（fund_basic）
- [ ] 历史净值数据补充（2018-2025年，~1,200个交易日/只）
- [ ] 历史持仓数据补充（2018Q4-2025Q4，~28个季度）

### 阶段2：指标计算（1-2周）
- [ ] 各区间收益计算（3m/6m/1y/2y/3y）
- [ ] 风险指标计算（波动率、回撤、夏普等）
- [ ] 同类排名计算（按8个类别分别计算）
- [ ] 质量评分模型建立（综合评分算法）

### 阶段3：数据验证（3-5天）
- [ ] 数据完整性检查（字段缺失情况）
- [ ] 异常值识别与处理（收益>100%或<-50%的核查）
- [ ] 指标回测验证（手工抽查10-20只基金验证计算正确性）

### 阶段4：系统上线
- [ ] 建立自动更新机制（Cron每日/双周更新）
- [ ] 建立数据质量监控（异常数据预警）
- [ ] 建立备份机制（定期备份数据库）

## 四、数据库建设完成后的启动流程

### Step 1: 数据质量验证（数据库完成后1-2天）
- 检查数据完整性（字段是否齐全）
- 验证指标计算准确性（抽样对比）
- 确认同类排名正确性（手工抽查几类基金）

### Step 2: 试运行筛选（第1周）
- 执行第一轮全市场筛选（按8个类别分别筛选前10%）
- 各类别选出5-10只基金进行快速分析
- 验证筛选标准和质量评分是否准确
- 调整阈值和权重（如需要）

### Step 3: 首批入库（第2-4周）
- 对各类别表现最优秀的2-3只基金进行深度分析
- 正式纳入核心库（目标：首批入库15-20只，覆盖8个类别）
- 建立产品档案模板

### Step 4: 正式运行（第2个月起）
- 双周滚动筛选
- 月度快速分析
- 季度深度入库
- 持续监控已入库基金表现

## 五、入库漏斗流程（每双周）

```
全市场基金 ~4,900只
    ↓ 按8个类别分别筛选前10%
候选基金 ~490只（各类别前10%）
    ↓ 一级指标筛选（排名+持续性+经理稳定性）
一级筛选后 ~50-80只
    ↓ 二级指标评估（夏普+回撤+波动率质量评分）
二级筛选后 ~20-30只
    ↓ 一票否决检查（经理变更/巨额赎回/违规等）
最终候选 ~15-20只
    ↓ 人工快速分析
月度分析 ~8-10只
    ↓ 季度深度+入库
季度入库 ~5-8只（分类型，核心库）
```
